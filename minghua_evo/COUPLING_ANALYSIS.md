# 茗花进化插件 (minghua_evo) - 解耦性评估报告

## 📋 摘要

本文档详细评估 **茗花进化插件 (minghua_evo)** 与原版项目的解耦程度，并提供完整的使用指南。

---

## 📦 两个独立软件

### 1. 茗花（原版软件）

**定位**：水果进货数据管理系统

**功能**：
- Excel 数据导入处理
- 报告生成（周报/月报/跨周对比）
- Web 端报告查看器（端口 8080）
- 数据 API 服务（端口 8081）

**独立运行**：`python3 src/report_viewer.py`

**包含组件**：
```
茗花/
├── src/
│   ├── data_loader.py    # 数据处理
│   ├── generators.py     # 报告生成
│   ├── data_api.py       # 数据 API 服务
│   └── templates/        # Web 前端
├── data/                 # 原始数据
└── reports/             # 生成的报告
```

---

### 2. 茗花进化插件（minghua_evo）

**定位**：通用智能对话助手框架

**功能**：
- 自然语言意图识别
- 对话历史记录
- 自动进化系统
- 智能对话 Web 界面（端口 3001）

**独立运行**：需要接入数据 API

**包含组件**：
```
minghua_evo/
├── core/
│   ├── intent_classifier.py    # 意图识别（通用）
│   ├── code_executor.py        # 指令执行（业务适配）
│   └── conversation_logger.py  # 对话记录（通用）
├── services/
│   ├── data_provider.py        # 数据客户端（通用 HTTP）
│   └── evolution_engine.py      # 进化引擎（通用）
├── api/
│   └── routes.py                # API 路由（业务适配）
├── config/
│   ├── settings.yaml            # 配置
│   └── permissions.yaml         # 权限配置
└── frontend/
    └── dist/                    # Web 前端
```

## ⚡ 耦合点分析

### 数据流

| 方向 | 内容 | 性质 |
|------|------|------|
| 插件 → 原版 | HTTP 请求 `GET /api/weekly` | 调用方，无代码依赖 |
| 原版 → 插件 | 无 | 单向 |

### 共享资源

| 资源 | 共享方式 | 影响 |
|------|----------|------|
| `data/` | 读取相同数据文件 | 可选（API 可独立） |
| `reports/` | 原版写入报告 | 插件不依赖，可选读取 |
| 端口 8081 | API 服务 | 插件依赖，但可配置 |

### 业务概念解耦

| 概念 | 耦合情况 | 说明 |
|------|----------|------|
| "报告" 文本 | ✅ 已移除 | 插件只返回结构化数据 |
| 店铺/水果/金额 | ⚠️ 业务数据模型 | 可通过配置适配 |
| 周/月/周对比 | ⚠️ 时间周期概念 | 可通过配置适配 |

## 🧩 分离影响分析

### 场景 A：原版离开插件

**影响：无** ✅

| 功能 | 状态 |
|------|------|
| 数据导入处理 | ✅ 正常 |
| 报告生成 | ✅ 正常 |
| Web 查看器 (:8080) | ✅ 正常 |
| 数据 API (:8081) | ❌ 停止（插件依赖） |

**结论**：原版是完整系统，进化插件是完全可选的增强功能。移除插件后，原版功能完全不受影响。

---

### 场景 B：插件离开原版

**影响：需要替代数据源** ⚠️

| 功能 | 状态 |
|------|------|
| 意图识别 | ✅ 正常 |
| 对话记录 | ✅ 正常 |
| 进化引擎 | ✅ 正常 |
| 数据查询 | ❌ 需要数据 API |

**结论**：插件可独立部署，只需接入符合规范的第三方数据 API。

## 🔄 第三方接入指南

### 接入要求

插件不关心具体业务，只要求数据 API 返回结构化 JSON 数据。

#### 1. 必须实现的端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/health` | GET | 健康检查，返回 `{"status": "ok"}` |
| `/api/weekly` | GET | 周数据查询 |
| `/api/stores` | GET | 店铺列表查询 |
| `/api/fruits` | GET | 水果列表查询（可选） |
| `/api/monthly` | GET | 月度数据（可选） |
| `/api/cross-week` | GET | 跨周数据（可选） |

## 🚀 插件使用指南

### 方式一：与原版配合使用

```bash
# 1. 启动原版数据 API
cd 茗花
python3 src/data_api.py &
# API 运行在 http://localhost:8081

# 2. 启动进化版
cd 茗花by_claw
python3 server.py &
# 访问 http://localhost:3001
```

### 方式二：独立部署（接入第三方 API）

```bash
# 1. 修改配置
vim minghua_evo/config/settings.yaml
# 设置 data_api.host 指向你的 API

# 2. 安装依赖
pip install fastapi uvicorn pyyaml

# 3. 启动
python3 -c "
from fastapi import FastAPI
from minghua_evo.api import router
import uvicorn

app = FastAPI()
app.include_router(router)
uvicorn.run(app, host='0.0.0.0', port=3001)
"
```

### 方式三：嵌入现有项目

```python
import sys
sys.path.insert(0, '/path/to/minghua_evo')

from fastapi import FastAPI
from minghua_evo.api import router
from minghua_evo.core import IntentClassifier, ConversationLogger

# 创建应用
app = FastAPI()

# 添加进化插件路由
app.include_router(router)

# 自定义意图（可选）
classifier = IntentClassifier()
classifier.add_rule(IntentType.CUSTOM, ['你的专属关键词'])

# 运行
app.run()
```

## ✅ 结论

### 解耦证明

1. **原版完全独立**：可移除插件独立运行，进化功能完全可选
2. **插件可复用**：只需接入符合规范的 HTTP API
3. **无业务耦合**：插件只处理结构化数据，不关心"报告"等业务概念
4. **门槛低**：新项目仅需实现数据 API 即可使用插件

### 复用条件

- ✅ 提供符合规范的 HTTP JSON API
- ✅ 配置 API 地址
- ✅ （可选）根据业务扩展意图规则
