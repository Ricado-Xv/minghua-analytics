# 茗花智能汇报系统 - 软件设计文档

## 1. 系统概述

### 1.1 目标
将茗花改造成一个**可自我进化的智能汇报Web应用**，通过聊天窗口接收用户指令，自动执行数据分析并展示结果，同时记录对话内容，由AI自动分析并生成产品设计，实现功能迭代。

### 1.2 核心特性
- 🗣️ **聊天式交互**: 自然语言输入，动态展示结果
- 🤖 **AI 外挂**: 集成 OpenClaw 作为智能大脑
- 🔄 **自我进化**: 记录对话 → AI 分析 → 自动改代码
- 🛡️ **安全分级**: 目录级代码修改权限控制

### 1.3 设计原则
- **最小侵入**: 尽可能少的改动原项目文件目录
- **通用可移植**: 进化系统作为独立模块，可复用到其他应用
- **解耦设计**: 聊天 UI、意图识别、进化引擎均可独立使用

## 2. 系统架构

本系统由**两个独立软件**组成，通过 HTTP API 解耦。

### 2.1 两个独立软件

#### 茗花（原版软件）
**定位**：水果进货数据管理系统

| 组件 | 说明 |
|------|------|
| 数据处理 | Excel 数据导入处理 |
| 报告生成 | 周报/月报/跨周对比 |
| Web 查看器 | 报告可视化（端口 8080） |
| 数据 API | JSON 数据服务（端口 8081） |

#### 茗花进化插件 (minghua_evo)
**定位**：通用智能对话助手框架

|------|------|
| 意图识别 | 自然语言理解 |
| 对话记录 | 历史记录存储 |
| 进化引擎 | 自动功能迭代 |
| 智能 Web | 对话界面（端口 3001） |

## 3. 模块设计

### 3.1 通用层模块（可独立复用）

#### 意图识别器 (IntentClassifier)
- **独立性**: ✅ 100%
- **可复用性**: 直接复用
- **功能**: 解析用户输入，判断用户意图

**识别策略**: 采用分级识别机制
1. **正则匹配（第一级）**: 使用预定义的规则表达式匹配用户输入
   - 优点：速度快，无需调用外部AI
   - 适用场景：明确、简单的指令（如"查看周报"、"本月情况"）
2. **AI 识别（第二级）**: 正则匹配失败时，调用 OpenClaw 进行语义理解
   - 优点：支持复杂、自然的语言表达
   - 适用场景：正则无法匹配的模糊输入

**识别流程**:
```
用户输入 → 正则匹配 → [成功] → 返回意图
           ↓ [失败]
        AI 识别 → 返回意图
```

**支持意图**:
- VIEW_WEEKLY_REPORT: 查看周报
- VIEW_MONTHLY_REPORT: 查看月度报告
- VIEW_CROSS_WEEK: 查看跨周对比
- QUERY_STORES: 查询店铺数据
- QUERY_FRUITS: 查询水果数据
- CUSTOM: 自定义需求

#### 对话记录器 (ConversationLogger)
- **独立性**: ✅ 100%
- **可复用性**: 直接复用
- **功能**: 记录所有用户对话，支持后续AI分析

#### 进化引擎 (EvolutionEngine)
- **独立性**: ✅ 100%
- **可复用性**: 直接复用
- **功能**: 分析对话，生成设计文档，自动修改代码，版本管理

#### 数据提供器 (DataProvider)
- **独立性**: ✅ 100%
- **可复用性**: 直接复用
- **功能**: HTTP 客户端，对接任意符合规范的 API

### 3.2 适配层模块（需业务适配）

#### 代码执行器 (CodeExecutor)
- **独立性**: ⚠️ 80%
- **可复用性**: 需适配业务 handler
- **功能**: 根据意图执行对应的代码逻辑

#### API 路由 (Routes)
- **独立性**: ⚠️ 80%
- **可复用性**: 需适配业务端点
- **功能**: FastAPI 路由定义

#### 前端界面 (Frontend)
- **独立性**: ⚠️ 60%
- **可复用性**: 需适配业务展示
- **功能**: Vue 3 智能对话界面

### 3.3 模块独立性评分

| 模块 | 独立性 | 可复用性 |
|------|--------|----------|
| `intent_classifier.py` | ✅ 100% | 直接复用 |
| `conversation_logger.py` | ✅ 100% | 直接复用 |
| `evolution_engine.py` | ✅ 100% | 直接复用 |
| `data_provider.py` | ✅ 100% | 直接复用 |
| `code_executor.py` | ⚠️ 80% | 需适配业务 handler |
| `routes.py` | ⚠️ 80% | 需适配业务端点 |
| `frontend/` | ⚠️ 60% | 需适配业务展示 |

**综合评分：9/10**

### 3.4 权限管理器 (PermissionManager)

**功能**: 管理代码修改的权限级别

**权限级别**:
- FORBIDDEN: 禁止修改
- CONFIRM_REQUIRED: 需用户确认
- AUTO: 自动修改

### 3.5 进化引擎 (EvolutionEngine)

**功能**: 分析对话，生成设计文档，自动修改代码，版本管理

**触发方式**:
- 手动触发：用户在界面点击"开始进化"按钮
- 定时触发：用户配置 Cron 表达式

**版本管理**:

## 5. API 设计

### 5.1 对外数据 API（原版 → 插件）

插件不关心具体业务，只要求数据 API 返回结构化 JSON 数据。

#### 必须实现的端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/health` | GET | 健康检查，返回 `{"status": "ok"}` |
| `/api/weekly` | GET | 周数据查询 |
| `/api/stores` | GET | 店铺列表查询 |
| `/api/fruits` | GET | 水果列表查询（可选） |
| `/api/monthly` | GET | 月度数据（可选） |
| `/api/cross-week` | GET | 跨周数据（可选） |

#### 响应格式规范

**健康检查**
```json
{"status": "ok"}
```

**周数据 (`/api/weekly`)**
```json
{
  "week": "第N周",
  "month": "N月",
  "summary": {
    "店铺数": 6,
    "水果种类": 8,
    "总进货量": 1156.1,
    "总进货额": 9865.35
  },
  "stores": [...],
  "fruit_stats": [...]
}
```

### 5.2 聊天 API（插件 → 用户）

**POST /api/chat**
```json
// Request
{ "message": "查看本月运营情况" }

// Response
{
  "intent": "VIEW_MONTHLY_REPORT",
  "reply": "为您生成3月月度汇总报告...",
  "data": { ... },
  "render_type": "chart"
}
```

### 5.3 进化引擎 API

- `POST /api/evolve/trigger` - 手动触发进化
- `GET /api/evolve/status` - 获取进化状态
- `GET /api/evolve/versions` - 获取版本历史
- `POST /api/evolve/schedule` - 设置定时任务

## 7. 进化功能扩展需求

### 7.1 意图识别优化
- **【重要】接入真正AI意图识别**：目前的意图识别是基于规则的正则匹配，需要接入OpenClaw实现真正的AI意图识别
  - 用户输入自然语言，AI理解真实意图
  - 支持更复杂的表达方式
  - 支持多轮对话上下文理解
- 优先级关注未识别文本
- **优先级关注未识别文本**：对于无法识别的用户输入（CUSTOM类型），优先记录并纳入进化分析
- **智能过滤闲聊**：对于闲聊类内容（如"你好"、"谢谢"、"辛苦了"等与业务无关的寒暄），不纳入功能设计参考
- **意图分类**：区分业务需求与闲聊，设置不同的处理策略

### 7.2 代码修改权限优化
- **白名单机制**：设置允许 AI 修改的目录白名单，白名单内的代码可被 AI 自动修改
- **黑名单机制**：关键核心文件（如配置文件、数据库连接等）加入黑名单，禁止任何 AI 修改
- **分级管理**：
  - 🔴 黑名单（禁止）：config.py、stores.json、fruits.json、数据库相关
  - 🟡 白名单（自动）：src/experiments/、src/plugins/
  - 🟢 待确认（默认）：其他业务代码

### 7.3 定时进化优化
- **定时任务独立运行**：开启定时进化后，不依赖用户输入日志，按时自动执行
- **进化内容来源**：
  - 近期用户对话（排除闲聊）
  - 已确认但未应用的设计文档
- **状态记录**：记录每次定时进化的执行时间、修改内容、结果

### 7.4 闲聊过滤机制
- **闲聊识别规则**：
  - 问候类："你好"、"早上好"、"晚安"
  - 感谢类："谢谢"、"辛苦了"
  - 肯定类："很好"、"不错"
  - 闲聊类："今天天气"
- **过滤策略**：上述类型不加入进化需求池，但仍记录在对话历史中
