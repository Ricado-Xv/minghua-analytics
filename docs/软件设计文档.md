# 茗花智能汇报系统 - 软件设计文档

## 1. 系统概述

### 1.1 目标
将茗花改造成一个**可自我进化的智能汇报Web应用**，通过聊天窗口接收用户指令，自动执行数据分析并展示结果，同时记录对话内容，由AI自动分析并生成产品设计，实现功能迭代。

### 1.2 核心特性
- 🗣️ **聊天式交互**: 自然语言输入，动态展示结果
- 🤖 **AI 外挂**: 集成 OpenClaw 作为智能大脑
- 🔄 **自我进化**: 记录对话 → AI 分析 → 自动改代码
- 🛡️ **安全分级**: 目录级代码修改权限控制

### 1.3 设计原则
- **最小侵入**: 尽可能少的改动原项目文件目录
- **通用可移植**: 进化系统作为独立模块，可复用到其他应用
- **解耦设计**: 聊天 UI、意图识别、进化引擎均可独立使用

## 2. 系统架构

本系统由**两个独立软件**组成，通过 HTTP API 解耦。

### 2.1 两个独立软件

#### 茗花（原版软件）
**定位**：水果进货数据管理系统

| Web 查看器 | 报告可视化（端口 8080） |
| 数据 API | JSON 数据服务（端口 8081） |

**定位**：通用智能对话助手框架

|------|------|

| 进化引擎 | 自动功能迭代 |

## 3. 模块设计

### 3.1 通用层模块（可独立复用）

#### 意图识别器 (IntentClassifier)
- **独立性**: ✅ 100%
- **可复用性**: 直接复用
- **功能**: 解析用户输入，判断用户意图

**识别策略**: 采用分级识别机制（详见第4节）

#### 对话记录器 (ConversationLogger)
- **独立性**: ✅ 100%
- **可复用性**: 直接复用
- **功能**: 记录所有用户对话，支持后续AI分析

#### 进化引擎 (EvolutionEngine)
- **独立性**: ✅ 100%
- **可复用性**: 直接复用
- **功能**: 分析对话，生成设计文档，自动修改代码，版本管理

#### 数据提供器 (DataProvider)
- **独立性**: ✅ 100%
- **可复用性**: 直接复用
- **功能**: HTTP 客户端，对接任意符合规范的 API

### 3.2 适配层模块（需业务适配）

#### 代码执行器 (CodeExecutor)
- **独立性**: ⚠️ 80%
- **可复用性**: 需适配业务 handler
- **功能**: 根据意图执行对应的代码逻辑

#### API 路由 (Routes)
- **独立性**: ⚠️ 80%
- **可复用性**: 需适配业务端点
- **功能**: FastAPI 路由定义

#### 前端界面 (Frontend)
- **独立性**: ⚠️ 60%
- **可复用性**: 需适配业务展示
- **功能**: Vue 3 智能对话界面

### 3.3 模块独立性评分

| 模块 | 独立性 | 可复用性 |
|------|--------|----------|
| `intent_classifier.py` | ✅ 100% | 直接复用 |
| `conversation_logger.py` | ✅ 100% | 直接复用 |

## 4. 意图识别模块详细设计

### 4.1 设计目标

意图识别模块的核心设计目标：

1. **高性能**：简单输入优先用正则，避开AI调用开销
2. **可进化**：规则与代码解耦，支持独立演进
3. **可反馈**：用户可纠正识别结果，改进识别准确度
4. **时间感知**：理解"上周"、"本月"等时间相关表达

### 4.2 分级识别流程

```
用户输入
    ↓
┌─────────────────────────────────────┐
│  第1层：闲聊过滤                    │
│  匹配 greetings 关键词               │
│  "你好" → 友好回复                  │
└─────────────────────────────────────┘
    ↓ (非闲聊)
┌─────────────────────────────────────┐
│  第2层：正则匹配                    │
│  匹配 intent_rules.yaml 规则         │
│  置信度 ≥ 0.8 → 直接返回           │
│  "周报" → VIEW_WEEKLY_REPORT       │
└─────────────────────────────────────┘
    ↓ (未匹配/低置信度)
┌─────────────────────────────────────┐
│  第3层：AI 识别                    │
│  One-shot 模式，无记忆              │
│  每次独立判断                       │
└─────────────────────────────────────┘
```

### 4.3 置信度体系

| 置信度 | 来源 | 说明 |
|--------|------|------|
| 1.0 | 闲聊关键词精确匹配 | "你好" → CUSTOM |
| 0.95 | 关键词完全匹配 | 用户只输入"周报" |
| 0.9 | 正则匹配成功 | "本周销售情况" |
| 0.7 | AI 识别 | 正则失败时调用 |
| 0.5 | 兜底 | 无法识别 |

**阈值配置** (`intent_rules.yaml`)：
```yaml
thresholds:
  high: 0.8   # 高于此直接返回
  low: 0.5    # 低于此走AI
```

### 4.4 时间上下文感知

模块内置当前时间上下文，支持智能时间词转换：

| 用户表达 | 实际解析 | 说明 |
|----------|----------|------|
| "上周" | month=N, week=N-1 | 自动计算上一周 |
| "本周" | month=N, week_offset=0 | 当前月 |
| "本月" | month=N | 当前月 |
| "上月" | month=N-1 | 上个月 |
| "下月" | month=N+1 | 下个月 |
| "3月第一周" | month=3, week=1 | 显式指定 |

当前时间通过 `datetime.now()` 实时获取，非写死。

### 4.5 规则配置解耦

所有识别规则存储在 `config/intent_rules.yaml`：

```yaml
rules:
  VIEW_WEEKLY_REPORT:
    keywords:           # 精确匹配（置信度 0.95）
      - 周报
      - 本周
      - 上周
      - 第一周
    patterns:          # 正则匹配（置信度 0.9）
      - "周报"
      - "本周.*销售"
      - "第.*周"

  VIEW_MONTHLY_REPORT:
    keywords:
      - 月报
      - 本月
    patterns:
      - "月报"
      - "本月.*情况"

# 闲聊过滤
greetings:
  - 你好
  - 您好
  - 谢谢
  ...
```

**可进化性**：进化引擎可直接修改此文件，无需改代码。

### 4.6 用户反馈机制

每次识别结果展示后，用户可选择：

| 操作 | 效果 |
|------|------|
| ✓ 正确 | 确认识别，记录反馈 |
| ✗ 不准 | 提交期望的提问方式 |

**反馈数据结构**：
```json
{
  "timestamp": "2026-03-17T23:40:32",
  "message": "3月第一周",
  "ai_intent": "CUSTOM",
  "user_input": "查看3月第一周数据",
  "comment": "应该识别为周报"
}
```

**存储位置**：`data/feedback/intent_feedback.json`

### 4.7 AI 识别实现

当正则匹配失败时，调用 OpenClaw 进行语义理解：

```python
# 使用唯一 session_id，每次全新会话（无记忆）
session_id = f"intent-{uuid.uuid4().hex[:8]}"

result = subprocess.run([
    "openclaw", "agent",
    "--agent", "test-agent",
    "--session-id", session_id,
    "-m", prompt,
    "--json"
], timeout=60)
```

**优点**：
- 每次独立判断，不受历史影响
- 不阻塞主对话
- 可配置独立 Agent

### 4.8 支持的意图类型

| 意图 | 说明 | 示例 |
|------|------|------|
| VIEW_WEEKLY_REPORT | 查看周报 | "查看本周销售" |
| VIEW_MONTHLY_REPORT | 查看月报 | "本月情况如何" |

| GENERATE_REPORT | 生成报告 | "生成报表" |
| CUSTOM | 自定义 | 其他需求 |

## 7. 进化功能扩展需求

### 7.1 意图识别优化
- **【重要】接入真正AI意图识别**：目前的意图识别是基于规则的正则匹配，需要接入OpenClaw实现真正的AI意图识别
  - 用户输入自然语言，AI理解真实意图
  - 支持更复杂的表达方式
  - 支持多轮对话上下文理解
- 优先级关注未识别文本
- **优先级关注未识别文本**：对于无法识别的用户输入（CUSTOM类型），优先记录并纳入进化分析
- **智能过滤闲聊**：对于闲聊类内容（如"你好"、"谢谢"、"辛苦了"等与业务无关的寒暄），不纳入功能设计参考
- **意图分类**：区分业务需求与闲聊，设置不同的处理策略

### 7.2 代码修改权限优化
- **白名单机制**：设置允许 AI 修改的目录白名单，白名单内的代码可被 AI 自动修改
- **黑名单机制**：关键核心文件（如配置文件、数据文件等）加入黑名单，禁止任何 AI 修改
- **分级管理**：
  - 🔴 黑名单（禁止）：`src/config.py`、`stores.json`、`fruits.json`、`src/report_viewer.py`
  - 🟡 待确认（默认）：`src/generators.py`、`src/weekly_report.py`、`src/data_loader.py`、`src/templates/`
  - 🟢 白名单（自动）：`minghua_evo/`（进化系统本身）

### 7.3 定时进化优化
- **定时任务独立运行**：开启定时进化后，不依赖用户输入日志，按时自动执行
- **进化内容来源**：
  - 近期用户对话（排除闲聊）
  - 已确认但未应用的设计文档
- **状态记录**：记录每次定时进化的执行时间、修改内容、结果

### 7.4 闲聊过滤机制
- **闲聊识别规则**：
  - 问候类："你好"、"早上好"、"晚安"
  - 感谢类："谢谢"、"辛苦了"
  - 肯定类："很好"、"不错"
  - 闲聊类："今天天气"
- **过滤策略**：上述类型不加入进化需求池，但仍记录在对话历史中

## 10. OpenClaw 集成

本节介绍如何在插件中集成 OpenClaw 作为智能大脑。

### 10.1 为什么使用 OpenClaw

| 方案 | 优点 | 缺点 |
|------|------|------|
| 直接调用 LLM API | 简单 | 需要管理 API Key |
| **OpenClaw CLI** | 无需 API Key、可复用自己的 agent | 需要 Gateway |
| OpenClaw HTTP API | 适合程序调用 | 部分工具默认禁用 |

本项目采用 **CLI 方式**，简单可靠。

### 10.2 调用方式

#### 方式一：CLI 调用（推荐）

```python
import subprocess

result = subprocess.run([
    "openclaw",
    "agent",
    "--agent", "test-agent",      # 使用的 agent
    "--session-id", session_id,   # 唯一 session（无记忆）
    "-m", prompt,                  # 提示词
    "--json"                       # JSON 输出
], capture_output=True, text=True, timeout=60)

# 解析返回
data = json.loads(result.stdout)
```

#### 方式二：HTTP API

```python
import requests

response = requests.post(
    "http://localhost:18789/tools/invoke",
    headers={"Authorization": "Bearer <token>"},
    json={
        "tool": "exec",
        "args": {"command": "openclaw agent -m '...' --json"}
    }
)
```

### 10.3 独立 Agent 配置

为避免阻塞主对话，建议创建独立的测试 agent：

```bash
# 在项目根目录执行
cd /path/to/茗花by_claw

# 创建 test-agent，workspace 指向插件目录
openclaw agents delete test-agent --force 2>/dev/null
openclaw agents add test-agent --workspace ./minghua_evo --model MiniMax-M2.5
```

**test-agent 配置说明：**
- workspace: `minghua_evo/` （插件目录）
- 可访问插件代码（当前目录）
- 可访问宿主项目（`../src/`）

### 10.4 配置项

在 `config/settings.yaml` 中配置：

```yaml
intent_classifier:
  use_ai: true              # 是否启用 AI 识别
  host: "http://127.0.0.1:18789"  # Gateway 地址
  agent: "test-agent"       # 使用的 agent
  timeout: 60               # 超时时间(秒)
```

### 10.5 One-shot 模式

为保证意图识别纯粹性，每次调用使用唯一 session_id：

```python
import uuid

session_id = f"intent-{uuid.uuid4().hex[:8]}"
# 每次都是全新会话，无记忆
```

优点：
- 每次独立判断，不受历史影响
- 不阻塞主对话
- 判断纯粹

### 10.6 注意事项

1. **Gateway 必须运行**：`openclaw gateway start`
2. **Agent 需要创建**：使用 `openclaw agents add` 创建
3. **超时处理**：建议设置 60s 超时
4. **错误处理**：CLI 可能超时或返回非零退出码
