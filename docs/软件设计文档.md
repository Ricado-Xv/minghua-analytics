**版本**：v1.3  
**日期**：2026-03-18

## 1. 系统概述

### 1.1 定位

茗花进化插件是一个**可自我进化的智能对话助手框架**，通过 HTTP API 与宿主系统通信，实现完全解耦。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 🤖 三级混合意图识别 | 闲聊过滤 → 正则匹配 → AI 识别 |
| 📝 用户反馈机制 | 识别错误 / 新需求 分类收集，支持 txt 文件上传 |
| 🔄 自我进化 | 从需求生成需求/设计文档 |
| ⚙️ 配置化数据源 | HTTP API / Mock 数据可切换 |
| 🖥️ 前后端解耦 | iframe 嵌入宿主系统页面 |
| 📖 宿主文档获取 | 自动获取宿主项目的 README 和设计文档 |

## 3. 核心模块

### 3.1 数据提供模块 (DataProvider)

**文件**：`services/data_provider.py`

```python
class DataProviderBase:
    """数据提供者基类"""
    DEFAULT_ENDPOINTS = {
        'health': '/api/health',
        'months': '/api/months',
        'weekly': '/api/weekly',
        'monthly': '/api/monthly',
        'cross_week': '/api/cross-week',
        'stores': '/api/stores',
        'fruits': '/api/fruits',
    }

class HTTPDataProvider(DataProviderBase):
    """HTTP API 数据源"""

class MockDataProvider(DataProviderBase):
    """Mock 数据源（测试用）"""
```

### 3.2 意图识别模块 (IntentClassifier)

**文件**：`core/intent_classifier.py`

```
用户输入
    ↓
┌───────────────────┐
│ 第1层: 闲聊过滤    │ → 匹配 greetings 关键词
└───────────────────┘
    ↓
┌───────────────────┐
│ 第2层: 正则匹配    │ → 匹配 intent_rules.yaml
└───────────────────┘
    ↓
┌───────────────────┐
│ 第3层: AI 识别    │ → OpenClaw Agent
└───────────────────┘
```

### 3.3 反馈收集模块 (FeedbackCollector)

**文件**：`core/feedback_collector.py`

**收集类型**：

| 类型 | 触发场景 | 存储 |
|------|---------|------|
| 意图反馈 | 用户点击"✗不准"→识别错误 | `intent_feedback.json` |
| 新需求 | 识别失败(CUSTOM)，或用户选择"新需求" | `requirements.json` |

**txt 文件支持**：用户可上传 txt 文件，每行作为一条需求

### 3.4 进化引擎模块 (EvolutionEngine)

**文件**：`services/evolution_engine.py`

```
用户需求 → AI 分析 → 生成需求文档 → 生成设计文档
```

### 3.5 文档生成模块

**文件**：
- `core/requirements_doc_generator.py` - 从反馈生成需求文档
- `core/doc_manager.py` - 文档版本管理

## 4. 接口设计

### 4.1 对话 API

```bash
POST /api/chat
{"message": "周报"}

# Response
{
  "intent": "VIEW_WEEKLY_REPORT",
  "reply": "为您查询...",
  "data": {...}
}
```

### 4.2 反馈 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/feedback/intent` | POST | 意图反馈 |
| `/api/feedback/intent` | GET | 反馈列表 |
| `/api/feedback/requirement` | POST | 新需求 |
| `/api/feedback/requirement` | GET | 需求列表 |
| `/api/feedback/requirements/batch` | POST | 批量提交需求 |

### 4.3 进化 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/evolve/generate-requirements` | POST | 生成需求文档 |
| `/api/evolve/generate-design` | POST | 生成设计文档 |
| `/api/evolve/progress` | GET | 进化进度 |

### 4.4 文档 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/docs/list` | GET | 文档列表 |
| `/api/docs/summary` | GET | 文档摘要 |
| `/api/docs/{filename}` | GET | 文档内容 |
| `/api/docs/status` | POST | 更新文档状态 |

### 4.5 配置 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/config` | GET | 系统配置 |
| `/api/project` | GET | 项目信息 |
| `/api/host-docs` | GET | 宿主文档 |

## 5. 配置设计

### 5.1 settings.yaml

```yaml
# 数据源配置
data_provider:
  type: "http"
  http:
    host: "localhost"
    port: 8081
    endpoints:
      health: "/api/health"
      weekly: "/api/weekly"
      # ...

# 宿主系统配置
host_system:
  name: "茗花"
  url: "http://localhost:3001"
  pages:
    VIEW_WEEKLY_REPORT: "/viewer.html?type=weekly"

# 进化配置
evolution:
  default_trigger: "manual"

# 服务配置
server:
  port: 3001
```

### 5.2 intent_rules.yaml

```yaml
rules:
  VIEW_WEEKLY_REPORT:
    keywords: [周报, 本周, 上周]
    patterns: [周报, 查看.*周.*数据]

greetings: [你好, 谢谢, 您好]

thresholds:
  high: 0.8
  low: 0.5
```
