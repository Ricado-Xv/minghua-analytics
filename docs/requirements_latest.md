## 1. 概述

### 1.1 产品背景

**茗花智能汇报系统** 是一个可自我进化的智能汇报 Web 应用，包含两大核心模块：

| 模块 | 定位 | 核心功能 |
|------|------|----------|
| 茗花（原版） | 水果进货数据管理系统 | Excel数据导入、周报/月报生成、跨周对比 |
| 茗花进化插件 | 智能对话助手框架 | 自然语言意图识别、用户反馈收集、自我进化 |

系统采用**三级混合意图识别模式**：

```
用户输入 → 闲聊过滤 → 正则匹配 → AI识别
```

意图类型包括：
- `VIEW_WEEKLY_REPORT` - 周报查询
- `VIEW_MONTHLY_REPORT` - 月报查询
- `VIEW_CROSS_WEEK` - 跨周对比
- `QUERY_STORES` - 店铺查询
- `QUERY_FRUITS` - 水果查询
- `GENERATE_REPORT` - 报告生成
- `CUSTOM` - 无法识别

### 1.2 产品目标

1. **提高意图识别准确率**：将核心业务意图识别准确率提升至 95% 以上
2. **扩大语义覆盖**：支持更多用户表达方式，包括动词、时间词、业务词的组合
3. **完善水果查询**：支持具体水果名称的查询
4. **优化用户体验**：减少用户纠错成本

### 1.3 目标用户

| 用户角色 | 典型场景 | 痛点 |
|---------|---------|------|
| 店长/店员 | "苹果卖了多少"、"查下上月销售" | 输入口语化，系统无法理解 |
| 区域经理 | "给我看看3月份的生意" | 业务词多样，系统识别困难 |
| 运营人员 | "查看3月第一周数据" | 跨月周报查询失败 |

### 2.2 根本原因分析

#### 2.2.1 规则层面

**问题1：动词前缀覆盖不足**

| 用户输入 | 包含动词 | 当前规则覆盖情况 |
|---------|---------|-----------------|
| 查看3月第一周数据 | 查看 | ✅ 有 `查看.*周.*数据` |
| 查下上月销售 | 查下 | ❌ 无 "查下" 动词 |
| 给我看看3月份的生意 | 看看 | ❌ 仅有 `看看.*周.*数据` |
| 苹果卖了多少 | 卖了 | ❌ 无 "卖了" 动词 |

**缺失的动词模式：**
- `查下`、`查一下`、`帮我查`
- `给我看看`、`帮忙看看`
- `卖了`、`卖了多少`、`卖得怎么样`

**问题2：时间词匹配不完整**

| 用户输入 | 时间表达 | 当前规则覆盖 |
|---------|---------|-------------|
| 查下上月销售 | 上月 | ❌ 无 "上月销售" 组合 |
| 给我看看3月份的生意 | 3月份 | ❌ 只有 "月份"，无 "3月份" |

**问题3：业务词（同义词）覆盖不足**

| 期望业务词 | 当前规则关键词 | 覆盖情况 |
|-----------|---------------|---------|
| 销售 | 数据、报表 | ❌ 缺失 "销售" |
| 生意 | 数据、报表 | ❌ 缺失 "生意" |
| 苹果、香蕉、橙子... | 水果 | ❌ 缺失具体水果名 |

**问题4：正则匹配逻辑问题**

当前 `VIEW_WEEKLY_REPORT` 的 pattern `查看.*周.*数据` 理论上可以匹配 "查看3月第一周数据"，但实际匹配失败，可能原因：
1. 正则贪婪匹配问题
2. `\\d+月第.*周` 与 `查看.*周.*数据` 优先级冲突

#### 2.2.2 架构层面

1. **同义词扩展机制缺失**：无法灵活配置"销售"、"生意"、"营收"为同义词
2. **业务词与意图绑定不灵活**：具体水果名无法自动触发 QUERY_FRUITS
3. **组合匹配权重不明**：动词+时间+业务词的组合置信度计算不清晰

### 2.3 影响范围

- 4条反馈中 **100%** 被错误识别为 CUSTOM
